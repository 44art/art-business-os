## 1. このメモを作成した理由

art-business-os は現在、**「ブランド登録 → AI分析 → ペルソナ作成 → コンテンツ/LP/LINE/SNS生成 → AIコンサルタント診断」** という売上導線支援ツールとして機能している。

将来的には、この導線を**広告宣伝・集客・CRM・LTV管理まで拡張**することが有力な方向性として考えられる。その際に参照できるよう、広告・成長系の考え方を今の段階で整理しておく。

---

## 2. 今は実装しない理由

| 理由 | 詳細 |
|------|------|
| **Ver.1のスコープ外** | Ver.1は「自分の導線を整理し、AI提案でたたき台を作る」ツール。広告運用自体はユーザーが行う |
| **外部API連携が必要** | Meta Ads API・Google Ads API・LINE Ads・Stripe等との連携が必要で、認証・コスト・セキュリティ要件が大きい |
| **CRM/MAは別システムが適切** | HubSpot・Klaviyo・Brevo等の専用ツールがすでに存在し、それらとの棲み分けが必要 |
| **UGC・紹介制度は運用設計が先** | 仕組みを作る前に、実際の販売実績と顧客層の確認が必要 |
| **LTV/CAC分析は決済データが必要** | Stripe・BASE・STORES等の決済データと連携しないと意味のある分析ができない |

## 3. 将来参考にする考え方

### 3-1. 広告宣伝の基本構造（アーティスト向け）

```
認知（広告/SNS/UGC）
  → 興味（コンテンツ/LP）
  → 信頼（ストーリー/作品世界観/実績）
  → 購入/申込（LP/DM/LINE）
  → リピート（LINE/SNS/メール）
  → 紹介（UGC/口コミ/紹介制度）
```

アーティストの場合、「信頼」フェーズが特に重要。  
作品や人柄・制作過程への共感が購入動機になるため、**広告よりコンテンツの質と継続性が効く**。

### 3-2. LTV / CAC の基本概念

- **LTV（Life Time Value）**: 顧客が生涯を通じてもたらす売上合計
  - 例: ワークショップ参加（5,500円）→ 作品購入（45,000円）→ カスタム依頼（35,000円+）
  - アーティストは1顧客あたりLTVが高い傾向（ファン化・ギフト需要）
- **CAC（Customer Acquisition Cost）**: 1顧客獲得にかかるコスト
  - 広告費 + 制作時間 + ツール費用 ÷ 獲得顧客数
- **LTV > CAC × 3** が持続可能な成長の目安

### 3-3. 広告媒体とアーティストとの相性

| 媒体 | 特性 | アーティスト向け用途 |
|------|------|---------------------|
| Instagram広告 | ビジュアル訴求が強い | 作品・WS体験の認知拡大 |
| Meta広告（Facebook） | ターゲティング精度が高い | リターゲティング・類似オーディエンス |
| TikTok広告 | 動画で世界観を伝えやすい | 制作過程・WS紹介の認知拡大 |
| LINE広告 | 既存LINE登録者へのリーチ | リピーター・ファン向け訴求 |
| Google広告（P-MAX） | 購買意図の高い検索流入 | 「スニーカーカスタム 東京」等の検索対応 |

### 3-4. LP設計とCVR改善の考え方

現在のLP作成支援で生成しているセクション構成は、広告→LP流入時も有効。  
将来追加を検討できる要素：

- **ヒートマップ分析**: どのセクションで離脱しているかの可視化
- **A/Bテスト**: ヘッドライン・CTA文言・画像の比較検証
- **広告→LP一貫性スコア**: 広告文言とLPキャッチコピーの整合性チェック
- **LPロード速度最適化**: Core Web Vitals対応

### 3-5. LINE公式アカウント活用の発展方向

現在のLINE活用支援では配信文のたたき台を生成している。  
将来参考にできる拡張概念：

- **ステップ配信の自動化**: 登録日からN日後に特定メッセージを自動送信
- **セグメント配信**: 購入履歴・属性別グループへの個別配信
- **友達紹介クーポン**: リピーター紹介による新規獲得
- **リッチメニューの設計**: 問い合わせ/作品一覧/WS申込への導線最適化

### 3-6. UGC（ユーザー生成コンテンツ）活用

アーティスト向けに特に有効なUGC戦略：

- **購入者の投稿をリポスト**: 作品を実際に使っている様子の認知拡大
- **WS参加者の作品紹介**: 「自分でも作れる」という信頼感の醸成
- **ハッシュタグ管理**: `#KENARTカスタム` 等のオリジナルタグを育てる
- **許諾ルールの整備**: DM許諾取得→リポスト運用フロー

### 3-7. 紹介制度の設計考え方

- **WS紹介割引**: 友人紹介で双方の次回WS参加費が割引
- **アフィリエイト的紹介**: インフルエンサー・コレクターに紹介リンク発行
- **ギフト需要の活用**: 「誰かへのプレゼント」特化のLPと紹介フロー

### 3-8. CRM / MA の概念

- **CRM（顧客関係管理）**: 購入履歴・接触履歴・好み等を管理し、適切なタイミングで適切なアプローチ
- **MA（マーケティングオートメーション）**: セグメント×タイミング×メッセージの自動化
- アーティスト規模では **LINE公式＋Googleスプレッドシート** が現実的な最初のCRM
- 将来的には **HubSpot無料プラン** や **Brevo（旧Sendinblue）** との連携も選択肢

以下は将来のバージョンで検討できる機能候補。実装優先度は事業状況によって判断する。

### 高優先度（売上直結）
- [ ] **広告訴求文の生成強化** — 既存のコンテンツ生成にInstagram広告・LINE広告向け文言を追加
- [ ] **LP CVR改善チェックリスト** — コンサルタント診断にLP評価軸を追加
- [ ] **LINE配信スケジュール提案** — ステップ配信の週次スケジュール案を生成

### 中優先度（継続利用促進）
- [ ] **LTV計算ツール** — 作品・WS単価と想定リピート率からLTV試算
- [ ] **CAC目安チェッカー** — 広告費入力でCAC目安と推奨LTV目標を計算
- [ ] **UGC活用アイデア生成** — ペルソナ別の口コミ促進施策案を生成

### 低優先度（将来統合）
- [ ] **Meta Ads API連携** — Facebook/Instagram広告の効果データ取込
- [ ] **Stripe決済連携** — 実売上データからLTV・リピート率を自動計算
- [ ] **LINE Messaging API連携** — 配信文を直接LINE公式から送信
- [ ] **Google Ads API連携** — 検索広告向けキーワード候補生成
- [ ] **紹介制度管理機能** — 紹介コード発行・追跡・割引管理

## 5. Ver.1に反映できる範囲（今後の改善で参照する）

広告機能を追加しなくても、**既存機能の提案品質向上**に活用できる考え方：

| 対象機能 | 活用できる概念 |
|----------|---------------|
| コンテンツ生成 | 広告コピーの構成（問題提起→共感→解決策→CTA）をSNS投稿文に反映 |
| LP作成支援 | 広告→LP一貫性（広告文とヒーローコピーの言葉を揃える）を評価項目に追加 |
| LINE活用支援 | ステップ配信の考え方（登録直後・3日後・7日後の流れ）を配信案に反映 |
| SNS戦略 | 投稿の「認知→信頼→行動」の3段階設計を投稿テーマ提案に活用 |
| AIコンサルタント | CAC・LTVの考え方を「費用対効果」の診断視点として追加 |

## 6. 実装しない機能の明確化（Ver.1範囲外）

以下の機能は Ver.1 では実装しない。

| 機能 | 理由 |
|------|------|
| Meta Ads / Google Ads / LINE Ads API連携 | 外部API認証・費用・セキュリティ要件が大きい |
| CRM機能（顧客管理DB） | 専用ツール（HubSpot等）との棲み分けが必要 |
| MA（マーケティングオートメーション） | 同上 |
| UGC収集・管理機能 | SNSプラットフォームAPIの利用規約・認証が必要 |
| 紹介制度・アフィリエイト管理 | 決済・追跡システムとの統合が必要 |
| LTV/CAC分析ダッシュボード | 実決済データ（Stripe等）との連携が必要 |
| A/Bテスト機能 | LP本番公開インフラが必要 |
| ヒートマップ・アナリティクス連携 | GA4・Hotjar等の外部サービスが必要 |

## 7. art-business-osの方向性メモ

**「広告宣伝を見据えた売上導線支援」がart-business-osの重要な方向性。**

現在のVer.1は「導線のたたき台をAIが作る」ツール。  
将来的には「広告費をかける前に、自然流入で検証した導線を広告でスケールする」ための支援ツールとして発展させることが一つの有力な方向性。

```
Ver.1（現在）：導線の設計・言語化支援
  ↓
Ver.2（候補）：導線の検証支援（CVR分析・改善提案）
  ↓
Ver.3（候補）：広告スケール支援（広告文生成・CACモニタリング）
  ↓
Ver.4（候補）：LTV最大化支援（CRM連携・MA設計・紹介制度）
```

このロードマップは確定ではなく、実際の利用状況・ユーザーニーズ・事業規模の変化を見ながら判断する。
